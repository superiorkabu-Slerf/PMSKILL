# Pressure Test

完成第一版分析后，必须切换到反方视角，主动寻找这个方向为什么会失败。

## 一、必问的七类致命问题

1. 用户真的会付费，还是只是口头认可
2. 需求是高频刚需，还是低频抱怨
3. 获客成本是否会高到吃掉利润空间
4. 产品价值能否在短时间内被用户感知
5. 团队是否缺少关键资源、渠道或专业门槛
6. 竞品或平台是否能快速复制并碾压
7. 合规、数据、交付或服务链路中是否存在硬性阻断

## 二、压力测试输出方式

至少列出：

- 生死线问题
- 为什么它危险
- 如果判断错误，会造成什么后果
- 可以怎样低成本验证
- 如果验证失败，项目该如何转向或停止

## 三、反方口吻建议

不要客气，也不要为了鼓励用户弱化问题。更合适的风格是：

- “如果这件事成立，最先应该出现什么证据？现在有吗？”
- “这是不是一个听起来痛，但不会驱动购买的场景？”
- “如果用户继续用 Excel、外包、人工流程，也能勉强解决，你凭什么切进去？”
- “如果只能服务很小的一群人，这个市场还能撑起什么规模？”

压力测试的目的不是否定一切，而是提前暴露最危险的假设。
