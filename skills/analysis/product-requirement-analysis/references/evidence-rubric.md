# Evidence Rubric

本 skill 的核心不是“写得像分析”，而是“尽可能用证据支撑判断”。

## 一、证据分层

### A 级：强证据

- 官方公开数据、财报、政策文件、平台规则
- 直接用户访谈记录、真实成交记录、实际使用数据
- 可追溯的行业报告或权威研究

### B 级：中证据

- 可信媒体报道
- 多个独立来源互相印证的市场信息
- 主流社区中重复出现的用户抱怨与需求信号

### C 级：弱证据

- 单篇观点文章
- 单条社交媒体评论
- 未经验证的行业传闻

## 二、输出时的标注要求

对每个关键结论，尽量标注以下之一：

- `事实`：已有较强证据支持
- `推断`：有部分证据，但仍需要验证
- `假设`：目前主要依赖逻辑判断，缺少外部证据

## 三、最低验证要求

如果要判断一个方向“值得继续”，至少要覆盖：

- 用户痛点信号
- 替代方案现状
- 市场或需求存在性
- 支付或预算可能性
- 一项主要风险

如果其中多数只能写成“假设”，结论只能是 `谨慎推进` 或 `暂不建议`。

## 四、竞品分析要求

竞品分析不要机械凑数量。每个竞品至少回答：

- 它主要服务谁
- 它解决什么问题
- 它怎么收费或如何获取用户
- 它最明显的短板是什么
- 我们如果进入，切口在哪里

## 五、禁止的证据错误

- 把用户说“有意思”当作付费意愿
- 把下载量或访问量直接当作 PMF
- 把“市场很大”当作“我们能切进去”
- 把竞品很多当作“需求一定成立”
- 把 AI 可实现性当作商业可成立性
